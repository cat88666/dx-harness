# 主题-线程与事件分发骨架

## 主题结论

你这套游戏后端不是“请求来了直接处理完”，而是“命令进入分组队列，再由事件和延迟任务继续推进状态”。`dx-game-core` 给出简化骨架，`dx-game-frame` 提供生产版增强实现。

## 核心链路

1. 命令进入 `ThreadManager`
2. 按 `userId` 或 `groupId` 进入 `TaskQueue`
3. `TaskQueue` 保证同组串行
4. 需要延迟时交给 `EventManager`
5. 延迟事件到时后再回到线程分发执行

## 两层实现的区别

- `dx-game-core`：便于理解，类少，逻辑直白
- `dx-game-frame`：加入更多分组模型、心跳分发、超时控制和生产运行能力

## 工作上的使用方式

- 读业务异常时，先判断是“命令没进队列”还是“事件没继续推进”
- 看性能问题时，先看分组粒度，再看线程池配置
- 看状态错乱时，优先怀疑同组串行边界是否被绕开
