# Query Rules

## 目标

让 `query / locate / runbook` 三类操作输出稳定，不退化成随意聊天。

## `/query`

适用：

- “大厅匹配失败通常沿哪条链路排查？”
- “某两个模块的职责边界是什么？”

要求：

1. 先找 `summaries/` 和 `synthesis/`
2. 再补 `entities/` 中的具体对象
3. 输出里必须给出至少一个可直达页面或源码对象

## `/locate`

适用：

- “定位大厅登录入口”
- “定位某个定时任务或核心链路”

要求：

1. 先命中 `entities/`
2. 再补相关 `runbooks/` 或 `summaries/`
3. 输出重点是“去哪看”，不是完整解释

## `/runbook`

适用：

- “用户进房失败怎么查？”
- “某个状态没有刷新怎么查？”

要求：

1. 直接返回检查顺序
2. 必须包含高概率原因
3. 必须给出关键源码入口
