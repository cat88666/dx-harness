# Prompt-Ingest-Deepen

用于 `/ingest-deepen <object-or-page>`。

## 输入

- 已存在的粗粒度页面或 `index.md` 可定位对象
- 本次要细化的模块、链路、任务、BUG、事故或 runbook 症状
- 可选：目标分支、工单号、日志、SQL 或复现描述

## 执行顺序

1. 先读 `index.md`，确认对象已经能定位到粗粒度页面。
2. 读取粗粒度页面，确认已有事实、分支和源码入口。
3. 只读取本次对象相关源码、配置、日志或工单，不做全仓扩散。
4. 优先更新已有页面；需要新建时按类型落到 `entities/`、`summaries/`、`changes/`、`runbooks/`。
5. 新增自然语言入口时更新 `index.md`。
6. 追加 `log.md`。

## 输出边界

- 一次只扩展一个对象或问题簇。
- 必须保留来源路径。
- 必须写明分支差异和不确定点。
- 不能把粗粒度仓库卡改成大而全的业务百科。

## 适合场景

- “继续展开德州蘑菇逃庄链路”
- “把大厅房间人数刷新链路细化成 runbook”
- “基于已有仓库卡补某个模块页”
- “根据一条事故日志沉淀排障步骤”
