# 任务-德州蘑菇逃庄需求补充

## 需求结论

在德州蘑菇游戏中，修复“上一次逃庄”的补充 BUG：玩家在结算后触发留座离桌时，如果该玩家正好处于下一手蘑菇庄位，不能留座离桌。

该需求属于 `dx-game-texas` 的牌桌内行为，不属于 `dx-game-hall`。

## 当前上下文

- 当前集成分支: `pre`
- 蘑菇开发分支族: `feture-sun-16603`、`origin/16603-final`、`origin/duik-mushroom-temp-final`
- 用户明确上下文: “当前德州分支正在开发和提测第三轮的蘑菇游戏开发和蘑菇逃庄需求补充”
- 关键目标: 用该 BUG 打通德州游戏开发 harness 流程，不只是修一个点

## 问题边界

现有留座离桌逻辑中：

- `USER_HOLD_SEAT` 入口先执行配置、人数、鱿鱼状态校验
- 玩家在手牌进行中或结算中会设置 `userItem.nextLeave = true`
- 结算后由后续流程调用 `UserHoldSeatHandler.immediatelyHoldSeatLeave(..., true)` 完成真正留座离桌
- 蘑菇逻辑在新一手开局时通过 `GameStartHandlerBase.doRoomGameStart` 选庄后调用 `mushRoomGameService.addMushRoom(tableId, buttonUser)`

补充需求要求在“结算后留座离桌”阶段增加蘑菇庄位约束：如果玩家是下一手蘑菇庄位，不能完成留座离桌，应取消 `nextLeave` 并提示。

## 推荐定位顺序

1. 先看 [[仓库-dx-game-texas]](../entities/仓库-dx-game-texas.md)，确认当前分支和蘑菇分支族。
2. 再看 [[主题-德州蘑菇逃庄留座离桌链路]](../summaries/主题-德州蘑菇逃庄留座离桌链路.md)，确认链路入口和候选拦截点。
3. 对比 `feture-sun-16603` / `origin/16603-final` 中 `UserHoldSeatCommand`、`UserHoldSeatHandler`、`MushRoomGameService`、`GameStartHandlerBase`。
4. 若要落代码，优先在结算后真正离桌前拦截，而不是只在用户点击 `USER_HOLD_SEAT` 时拦截。

## 验收要点

- 蘑菇开关未开启时，原留座离桌行为不变。
- 非下一手蘑菇庄位玩家，原留座离桌行为不变。
- 玩家在结算后被标记 `nextLeave = true`，但若下一手是蘑菇庄位，应不能移动到占座玩家列表。
- 拦截后需要清理或回滚 `nextLeave`，避免下一轮重复触发。
- 提示优先复用 `TexasToastCode.TEXAS_MUSHROOM_NEXT_BUTTON_CANNOT_LEAVE`，该 toast 在 `dx-game-frame` 中已存在。
- 需要覆盖结算后路径，不只覆盖主动点击时的前置检查。
